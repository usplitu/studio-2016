Presentation and question & answer session

Introduction

As discussed during the studio sesssion, the final act in the grading process will comprise a presentation by you of your completed application and a question and answer session related to the application, JavaScript and the Git versioning system.

The questions will be based on the following list.

As indicated in the schedule already published, the total time allotted for each student is 20 minutes, 10 minutes for presentation and the remaining 10 for Q&A.

Play Framework

  1. What is a framework?
  2. What is the Play Framework?
  3. You have been developing using version 1.x. There is a later version 2.x. What are some fundamental differences? Would your version work without refactoring were you to upgrade to Play 2?
  4. Play uses the Model View Controller (MVC) pattern. Briefly explain the pattern. Why is it used? Is it unique to Play?
  5. Each Play application contains a routes file. What is its purpose?
  6. Here is a typical extract from a routes file. Explain.
    # Donation page
GET    /donation                                DonationController.index
POST   /donation/donate                         DonationController.donate
  7. What is the purpose of the application.conf file?
  8. The following Play command is frequently used. What is the purpose of doing so?
    play eclipsify
  9. Play uses a templating engine. Explain. Discuss where templating has been used in your project application, for example.
  10. What is the role of the controller?
  11. What is the role of the view?
  12. What is the role of the model?
  13. Explain @Entity and the parent class Model as used in this code snippet:
    @Entity
public class Donation extends Model 
{
...
}
  14. Explain the following terms:
    @ManyToOne
@OneToMany
@OneToOne

JavaScript Language (JS)

  1. Who developed the JavaScript language and when was it developed?
  2. What have Java and JavaScript in common (other than the word Java)?
  3. What is ECMAScript? How does it compare to JavaScript?
  4. What role does JavaScript play in developing client-side code?
  5. What is ES6? What preceded it? Name some new features introduced in ES6.
  6. JS now has 6 primitive types. What is a primitive type? List some of the six.
  7. JS is a loosely typed language. Explain.
  8. Discuss any behavioural differences between the types var, const and let.
  9. Java has several number types, for example, int, long, float and so on. How many number types does JS have?
  10. Discuss the strengths and weaknessess of the JS number types.
  11. What is the difference between null and undefined?
  12. ES6 introdued the Symbol type. What is it and what is its purpose?
  13. What is a JS object?
  14. What is an object literal?
  15. How do JS objects and arrays differ? When would one use an object, when an array?
  16. Discuss the use of semi-colons in JS.
  17. Discuss the differences, if any, between the uses of the equality operators in the following code:
    String s1 = 'Hello world';
String s2 = 'Hello world';
let b1 = s1 == s2;
let b2 = s1 === s2;
  18. Explain the terms truthy and falsy.
  19. Explain how the use of the && & || operators differ in JS compared to in Java.
  20. How is control flow achieved in JS?
  21. Where may one place JS code in a web app? Discuss the implications of where such code is located?
  22. How might one simultaneously debug both client JS and Play server Java code?
  23. Explain the terms function and method.
  24. What is an anonymous JS function?
  25. Discuss some of the implications in using strict mode.
  26. What is the arrow function? How does the behaviour of the this object or reference change when using an arrow function?
  27. A JS function is sometimes referred to as a first class object. Explain. In what way, if any, does it differ from a regular JS object?
  28. jQuery: what is it? Why use it?
  29. Explain:
    • The jQuery function,
    • A jQuery function,
    • A jQuery method,
    • A jQuery object.
  30. In the context of a JS file, what do the terms minification and obfuscation mean and why might these processes be used?
  31. What are the options available and steps involved to introduce jQuery into an application?
  32. JS is referred to as a prototypal language. Explain. Compare to Java.
  33. What does the term scope mean? How do scopes differ in the JS and Java languages?
  34. Explain and discuss in the context of JS the terms global scope, function scope and block scope.
  35. What is an immediately invoked function expression (IIFE)? When might one use an IIFE?
  36. Describe some techiques to avoid polluting the global scope (namespace)?
  37. Explain Temporal Dead Zone (TDZ).
  38. Explain what is meant by a closure. Describe an example where use of a closure would be advantageous.
  39. Explain what is meant by Ajax (Asynchronous Java + XML). Why might one use it?

Git Versioning System

  1. Git is a distributed versioning system. How does this compare to centralized version control systems?
  2. Describe in broad terms how to put an existing application under Git version control.
  3. Distinguish between client-side and server-side Git, providing examples.
  4. Would you put all files in a project under Git control? If not, why not? How would decide which files to exclude and how would you achieve this?
  5. What do you hope will happen when you run this command?
    git push origin master
  6. What do you hope will happen when you run this command?
    git pull origin master
  7. What is the index?
  8. What does the commit command achieve?
  9. What are the consequences for you working folder and its contents when, while on tht master branch, you run a command such as the following. What might dev refer to?
    git checkout dev
  10. How many object types in Git, list them and briefly explain?
  11. What is SHA-1, a SHA-1 hash and what role do they play in Git?
  12. The merge and rebase commands may be used, for example, to move a development branch to the master branch. Which command should one use and why?
  13. How might one revise the commit history of a project, for example in reducing the number of commit messages or changing the message content of a message or messages? What is good practice in doing so?
